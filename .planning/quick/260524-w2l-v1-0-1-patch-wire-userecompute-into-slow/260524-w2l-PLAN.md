From frontend/src/hooks/useRecompute.ts:
```typescript
export function useRecompute(): {
  triggerRecompute: (changedParams: Record<string, number>) => Promise<void>
  triggerRetrain: (params: { vector_size: number; window: number }) => Promise<void>
  cancelRecompute: () => void
}
```

The hook already:
- Calls `store.setIsRecomputing(true)` / `store.setIsRetraining(true)` internally.
- Sends `POST /viz/recompute` with `{ changed_params }` via `apiFetch`.
- Opens a WebSocket on `WS_BASE/ws/recompute/{job_id}` for progress.
- Clears the flag + dirty params and invalidates React Query keys on `done` / on error.

So the consumer's job is ONLY: import the hook, call the right method, drop `console.log`
and drop manual `setIsRetraining(true)` (the hook owns that flag now).

From frontend/src/components/settings/SlowTierParams.tsx (current handleRecompute, lines 59-68):
```typescript
const handleRecompute = useCallback(() => {
  const changed: Record<string, number> = {}
  for (const key of dirtyParams) {
    if (values[key] !== undefined) changed[key] = values[key]
  }
  // Trigger recompute via useRecompute hook (wired by parent)
  // For now just log - actual wiring uses useRecompute
  console.log('Recompute triggered with:', changed)
}, [dirtyParams, values])
```

From frontend/src/components/settings/VerySlowTierParams.tsx (current handleConfirmRetrain, lines 40-45):

const handleConfirmRetrain = useCallback(() => {
