AB · 07

Empty C · Classification failure

sidebar zoomlight · Paper3 specific variants

Never "Error occurred." Each failure mode gets its own headline pattern — "We couldn't [verb] this file" — plus one clear next action. The 410-Gone (5-min Redis TTL) and 503 (calibration missing) variants are shown below since they're the two that most users hit when a job stalls.

PRIMARY · ENCODING ERROR
UPLOAD & CLASSIFY
We couldn't read this file
Looks like the encoding wasn't UTF-8. Try saving the file as plain UTF-8 text and dropping it again.
VARIANTS
410 GONE
This result expired.
Upload the file again to recompute. (5-min Redis TTL on feature_vec.)
503 · CALIBRATION
Top genre still shown.
Probability bars degraded. The classifier is fine; only the calibration step missed.
< 500 WORDS
Too short to classify.
Try a longer book — at least 500 words.
WRONG FORMAT
Only .txt files for now.
PDF and EPUB support is on the v3 roadmap.
DESIGN INTENT
HEADLINE PATTERN
"We couldn't [verb] this file"
Apologetic + specific. Never "Error occurred" or "Something went wrong."
TWO COLOR LEVELS
Red wash for "you need to fix this" (encoding, too short, wrong format). Amber wash for "we'll handle this" (410 expired, 503 calibration). Different action ladder.
ONE NEXT ACTION
Every variant offers exactly one button or one explicit re-try path. No "Contact support."
What stays The classify input remains visible above the error so the user can try a new file without navigating away.
410 framing
"Expired", not "Lost". The user understands TTL implicitly when the language is concrete.
503 framing
Top-1 prediction still rendered with no probability bars. The error sits next to the partial result, not instead of it.
Future-proofing
Wrong-format card pre-mentions v3 PDF/EPUB to set the right expectation. Costs nothing now.
