# Plan 10-01 — Visual Polish full sweep

## Objective

Implement Phase 10 end-to-end per `design_handoff/.../README.md` §15 — add light/dark/system theming with a new persistent `preferencesStore`, refactor the ~30 inline-hex Phase 9 components to HSL CSS variables, swap to the v2 dual-token genre palette, wire scene background to update imperatively without canvas remount, add a header `?` Help dropdown with 3-state theme segmented control, create a hand-rolled 4-step onboarding tour anchored on stable `data-tour-id`s centralised in `src/tour/anchors.ts`, polish the four empty states (upload zone, Compare, classification failure, ClassificationExplain pre-upload), and land a Playwright smoke test that iterates `TOUR_STEPS` against a freshly mounted app.

Reference spec: `.planning/phases/10-visual-polish/design_handoff/design_handoff_phase10_visual_polish/README.md` (682 lines) — this is the canonical source of truth for every token value, copy string, layout decision, and pitfall.
