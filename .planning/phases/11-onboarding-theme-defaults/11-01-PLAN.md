# Plan 11-01 — Onboarding & Theme Defaults

## Objective

Make light the default theme for new visitors and build the first-visit onboarding sequence: auto-open "How It Works" on first visit (or first in ≥30 days), then chain into the existing 4-step tour when it closes. Persist the re-trigger via an `introSeenAt` timestamp. Reverses two Phase 10 decisions (D-58 dark-default, D-73 tour-auto-start) per the user's 2026-05-28 request.

Reference: `.planning/phases/11-onboarding-theme-defaults/11-CONTEXT.md` (decisions D-86..D-90 + exact inline script + edge cases).

## Tasks (atomic commits, in order)

### Task 1 — Light default + no-FOUC first paint (ONBOARD-01 / D-86)
- `frontend/src/stores/preferencesStore.ts`: change default `theme: 'system'` → `theme: 'light'`.
- `frontend/index.html`: add the inline pre-hydration `<script>` from 11-CONTEXT.md `<specifics>` into `<head>` BEFORE the module bundle. It reads `localStorage['lgt-prefs-v1']`, resolves the effective theme (new user → light), and adds `.light` to `<html>` so first paint is light with no dark flash.
- Verify: a fresh browser (cleared localStorage) loads in light mode with no dark flicker; an existing dark/system user is unaffected.
- Commit: `feat(11-01): default new users to light theme with no-FOUC first paint (D-86)`

### Task 2 — `introSeenAt` timestamp in preferencesStore (ONBOARD-02 / D-87)
- Add `introSeenAt: number | null` (default `null`) + `setIntroSeenAt(ts: number | null)` action to `preferencesStore`.
- Keep `tourCompleted` (still used for the tour's own state / nothing auto-starts off it now, but leave the field for compatibility + future use).
- Bump persist behavior: the new field rides the existing `lgt-prefs-v1` key. (Zustand persist tolerates added fields; missing `introSeenAt` in old payloads reads as `undefined` → treat as null.)
- Export `INTRO_TTL_MS = 30 * 24 * 60 * 60 * 1000` and a helper `isIntroStale(introSeenAt: number | null): boolean` returning `introSeenAt == null || Date.now() - introSeenAt >= INTRO_TTL_MS`.
- Commit: `feat(11-01): add introSeenAt timestamp + 30-day staleness helper (D-87)`

### Task 3 — Stop the tour auto-starting independently (D-89)
- `frontend/src/tour/TourProvider.tsx`: remove the `useEffect` that auto-starts the tour when `tourCompleted === false`. Keep `start()`, `next()`, `prev()`, `skip()`, `done()` and the `useTour()` hook intact for the chain + manual replay.
- Ensure `HelpDropdown`'s "Replay tour" → `tour.start()` still works (no change expected, just verify).
- Commit: `refactor(11-01): tour no longer auto-starts on first load (D-89)`

### Task 4 — PipelineExplanation onClose hook (D-88/D-90 plumbing)
- `frontend/src/components/explanation/PipelineExplanation.tsx`: ensure closing the walkthrough (X button, Esc, outside-click — every close path) invokes a single close handler. If it already routes all closes through `setPipelineExplanationOpen(false)`, that's the signal App will observe; otherwise add an optional `onClose?: () => void` prop called on every dismissal. Prefer observing the `pipelineExplanationOpen` transition in App (true→false) to avoid prop drilling — but if the component manages internal close state, expose the hook.
- Do NOT change any walkthrough content (Steps 1–7 stay verbatim — Phase 10 §12 NOT-list still applies).
- Commit: `feat(11-01): expose How It Works close signal for onboarding chain`

### Task 5 — Onboarding orchestration in App (ONBOARD-02/03 / D-88/D-90)
- `frontend/src/App.tsx`:
  - On mount (once), read `introSeenAt`; if `isIntroStale(introSeenAt)`:
    1. `setPipelineExplanationOpen(true)`
    2. set an `introSequenceActive` ref/state to `true`
    3. `setIntroSeenAt(Date.now())` (consume-on-fire so reload mid-sequence doesn't reopen — D-88)
  - Observe How It Works closing: when `pipelineExplanationOpen` transitions true→false AND `introSequenceActive` is true → clear `introSequenceActive` and call `tour.start()` (chain). Use a small delay (~300ms) so the modal unmount settles before the tour measures anchors.
  - Manual opens (nav button / Help dropdown) set `pipelineExplanationOpen` without touching `introSequenceActive`, so closing them does NOT start the tour (D-90).
  - Guard the mount effect so it runs exactly once (ref latch), not on every render.
- Verify the full chain: fresh load → How It Works opens → close it → tour runs → finish/skip → reload → neither reopens (within 30 days).
- Commit: `feat(11-01): first-visit How-It-Works→tour onboarding chain (ONBOARD-02/03)`
