## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → build_corpus.py | Untrusted text fetched from Project Gutenberg (HTTPS) crosses into our corpus |
| build_corpus.py → corpus/books.yaml + data/raw/ | Fetched bytes + computed metadata become committed corpus state |
| Wave-1 commit → Wave 2 retrain | books.yaml + raw text become inputs to Word2Vec training |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | Gutenberg URL fetch (download_book) | mitigate | Use HTTPS URLs only (gutenberg.org/cache/epub/{id}/pg{id}.txt + fallbacks); reject status≠200 or body<1000 bytes per 01_download_corpus.py pattern; compute text_sha256 over canonical post-strip bytes |
| T-08-02 | Integrity | text_sha256 vs data/raw file mismatch | mitigate | Acceptance criterion: post-build verification step recomputes sha256 of every data/raw/<id>.txt and asserts equality with corpus/books.yaml source.text_sha256 |
| T-08-03 | Repudiation / provenance forgery | source.{provider,fetched_at,text_sha256} field | mitigate | Schema-required all three subfields; build_corpus.py rejects emission if any is empty; manual edits forbidden post-Wave-1 (D-26) |
| T-08-04 | Tampering — cache poisoning by stale artifact | data/cache/ holding pre-v2 entries | mitigate | D-25 pre-retrain smoke test: flush_v1_cache.py --dry-run + a precompute lookup must show zero stale-keyed entries before commit |
| T-08-05 | Information disclosure (code injection via fetched text) | strip_headers output → tokenizer | accept | No eval/exec on fetched text in the pipeline; downstream consumers (preprocess.py, train_embeddings.py) treat text as opaque byte stream |
