# Plan 01-04: Pipeline Refinement — Corpus Expansion, Architecture Fixes, Parameter Sweep

**Phase:** 01 -- Pipeline Validation Spike
**Goal:** Improve classification accuracy and topological signal quality by: expanding the corpus to 100 books across 10 genres; fixing the persistence image to always output a stable 400D vector with independent per-axis resolution; dropping PCA in favour of direct kernel SVM with VarianceThreshold; adding class_weight='balanced'; sweeping Word2Vec window size (4 values), K-means cluster count (8 values), alpha (9 values), and SVM C (3 values) to find the optimal configuration.
**Status:** in-progress

---

## Architecture Reference

```
Word2Vec (150D, shared, trained on all books)
    ↓ per window size w ∈ {5, 10, 15, 20}
TF-IDF weights (per book, max-normalised to [0,1])
    ↓
Weighted Vietoris-Rips: D_ij = ||v_i - v_j|| / (w_i + w_j)
    ↓
H1 persistence diagram (birth, death) pairs
    ↓ rotate → (birth, persistence), weight by persistence
Persistence image: 20×20 = 400D
  - Global birth axis: [birth_min, birth_max] / 20 equal bins
  - Global pers axis:  [pers_min,  pers_max]  / 20 equal bins  ← independent
  - Gaussian kernel σ=0.05 at each (b_i, p_i), weighted by p_i
  - Always 400D regardless of axis length mismatch

K-means (K clusters, fit on all word vectors in 150D space)
    ↓
Cluster histogram: K-D, TF-IDF-weighted hard assignment, L2-normalised

Feature = [ α × topology_norm (400D) | (1-α) × location_norm (K-D) ]  ∈ ℝ^(400+K)

StandardScaler → VarianceThreshold(1e-4) → SVC(RBF, class_weight='balanced')
```

**What changed vs 01-03:**
- H0 dropped (birth axis collapses to 0 in Vietoris-Rips — confirmed degenerate)
- Persistence image: custom 400D output replacing persim's non-square output
- PCA dropped: class-blind, consumes all DOF at n=15 (14-component PCA on 14 training points)
- VarianceThreshold replaces PCA: removes always-zero TDA pixels without consuming DOF
- class_weight='balanced': forces SVM to respect minority/zero-accuracy genres (horror was 0%)
- Multi-window Word2Vec: sweep w ∈ {5,10,15,20} to find optimal semantic granularity
- 150D embeddings (was 100D)
- min_count=2 (was 5): lower threshold, TF-IDF handles downweighting of rare terms
- Corpus: 100 books, 10 genres, 10 per genre (was 15 books, 3 genres, 5 per genre)

### Task 01-04-05: Rewrite script 05 — custom 400D persistence image

**Goal:** Replace persim's non-square output with a custom 400D persistence image using independent per-axis 20-bin resolution. Add `--window` and `--k-clusters` args. Save per-(window, k) feature matrices.
**Requires:** Task 01-04-04

<implementation>

**`scripts/05_build_features.py`:**

**Custom persistence image function (replaces persim transform):**
```python
def make_persistence_image(diagrams_list, grid_resolution, sigma):
    """
    diagrams_list: list of (n_i, 2) arrays in (birth, persistence) coords.
    Returns: (imager_meta, transform_fn) where transform_fn maps one diagram -> 400D vector.

    Grid is defined globally across all books:
      birth axis: [b_min, b_max] / grid_resolution equal bins
      pers axis:  [p_min, p_max] / grid_resolution equal bins  (independent)
    Each point (b_i, p_i) contributes w_i=p_i weighted Gaussian with bandwidth sigma.
    """
    # 1. Compute global axis ranges
    all_births = np.concatenate([d[:, 0] for d in diagrams_list if len(d) > 0])
    all_pers   = np.concatenate([d[:, 1] for d in diagrams_list if len(d) > 0])
    b_min, b_max = all_births.min(), all_births.max()
    p_min, p_max = all_pers.min(),   all_pers.max()

    # 2. Build 20x20 grid centres
    b_centres = np.linspace(b_min, b_max, grid_resolution)  # shape (20,)
    p_centres = np.linspace(p_min, p_max, grid_resolution)  # shape (20,)
    B, P = np.meshgrid(b_centres, p_centres, indexing='ij')  # (20, 20) each
    grid = np.stack([B.ravel(), P.ravel()], axis=1)  # (400, 2)

    def transform(diagram):
        """diagram: (n, 2) array of (birth, persistence) pairs. Returns 400D vector."""
        if len(diagram) == 0:
            return np.zeros(grid_resolution ** 2)
        births = diagram[:, 0]
        perss  = diagram[:, 1]
        weights = perss  # weight by persistence (longer-lived = more significant)
        # Evaluate weighted Gaussian sum at each of 400 grid points
        # img[j] = sum_i w_i * N(grid[j] | (b_i, p_i), sigma^2 I)
        diff = grid[:, np.newaxis, :] - np.stack([births, perss], axis=1)[np.newaxis, :, :]
        # diff shape: (400, n, 2)
        sq_dist = np.sum(diff ** 2, axis=2)  # (400, n)
        gaussians = np.exp(-sq_dist / (2 * sigma ** 2))  # (400, n)
        img = gaussians @ weights  # (400,)
        return img

    return transform
```

**Per-book pipeline:**
1. Load `diagrams_{gid}_w{window}.npy` for all 100 books → convert to (birth, persistence) coords.
2. Call `make_persistence_image` with all H1 diagrams → get `transform_fn`.
3. For each book: apply `transform_fn` → 400D vector → L2-normalise.
4. Load K-means on word vectors for this window (or fit fresh if not cached).
5. Compute K-D cluster histogram → L2-normalise.
6. Concatenate: `feature = np.concatenate([alpha * topo_norm, (1-alpha) * loc_norm])` → (400+K)-D.
7. Save `data/features/feature_matrix_w{window}_k{k}.npy` and `data/features/labels.npy` (same across all configs), `data/features/book_order.json`.

**CLI args:** `--window W --k-clusters K --alpha A`
(alpha is applied at step 6; default 0.7 but sweep passes each value)

**K-means caching:** Save `data/models/kmeans_w{window}_k{k}.pkl` after fitting. If file exists, load instead of refitting (K-means is the same for a given window+k regardless of alpha).

</implementation>

```python
WINDOWS = [5, 10, 15, 20]
K_VALUES = [20, 30, 40, 50, 60, 70, 85, 100]

```

**Execution structure:**
```
results = []
for window in tqdm(WINDOWS, desc="Windows"):
    run("python scripts/03_train_embeddings.py --window {window}")
    run("python scripts/04_compute_homology.py --window {window}")

    for k in tqdm(K_VALUES, desc="K", leave=False):
        run("python scripts/05_build_features.py --window {window} --k-clusters {k}")
        # alpha applied in script 06, not 05 -- no re-run of 05 for each alpha

            for C in C_VALUES:
                result = run_validate(window, k, alpha, C)
                results.append(result)
                append_to_csv(results_dir / 'sweep_results.csv', result)
```

**Progress:** Outer tqdm over windows (4), inner nested bars showing k/alpha/C progress. Estimated total time printed at start based on benchmark.

**Resume support:** If `sweep_results.csv` exists and a given `(window, k, alpha, C)` row is already present, skip that combination. This allows resuming interrupted sweeps.

## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| VR computation time with 100 books × 4 windows | Medium | In-process VR (no subprocess overhead); 100 books × ~1s avg = ~100s per window; 4 windows = ~7 min total |
| Gutenberg download failures for some IDs | Medium | Resume-safe download (skip existing); retry loop in script 01; failures logged clearly |
| K-means non-determinism across sweep runs | Low | `random_state=42` fixed; K-means cached per (window, k) so fitted once |
| Persistence image empty for some books | Low | Empty diagram → zero 400D vector; VarianceThreshold will remove these features if they're always zero |
| Sweep interrupted mid-run | Low | Resume support in script 07 via CSV row presence check |
| Non-nested CV optimism if C selected by same LOOCV used for accuracy | Medium | At 100 books, using fixed C grid {1,10,100} and reporting all three — no selection bias. Nested CV documented as next step if further tuning needed |
| class_weight='balanced' may reduce overall accuracy while improving per-class | Low | Both balanced and unbalanced results in sweep CSV; report both; primary metric is balanced |
