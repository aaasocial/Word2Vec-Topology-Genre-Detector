# Plan 01-01: Corpus Assembly and Text Processing Pipeline

**Status:** complete
**Completed:** 2026-04-12
**Phase:** 01 -- Pipeline Validation Spike
**Goal:** Download 15 public domain books from Project Gutenberg, preprocess text (tokenize, normalize, filter stopwords), enforce minimum word count, and store processed output for downstream pipeline stages.
**Requirements:** CORPUS-01, CORPUS-04, PIPE-01, PIPE-02, CORPUS-03

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Arbitrary text content from Gutenberg (encoding errors, unexpected bytes) | Low | UTF-8 decode with `errors='replace'`; never `eval()` text content |
| Path traversal via book IDs in file paths | Low | Validate all Gutenberg IDs are integers before constructing file paths |
| Gutenberg rate limiting / IP ban | Low | Enforce 2-second sleep between downloads per D-09; cache downloads to `data/raw/` |
| Corrupted/truncated downloads | Low | Verify downloaded text length > 1000 chars before saving; re-download on failure |
</threat_model>

### Task 01-01-01: Download corpus from Project Gutenberg

**Goal:** Implement `scripts/01_download_corpus.py` that reads `corpus/books.yaml`, downloads each book via gutenbergpy, strips Gutenberg headers/footers, and saves raw text to `data/raw/{gutenberg_id}.txt`. Per D-06, print verbose timing for each download. Per D-12, warn and continue on download failures.
**Requires:** Task 01-01-00 (params.yaml, books.yaml, directory structure, utils.py)

<implementation>

**Create `scripts/01_download_corpus.py`:**

1. Load `corpus/books.yaml` with PyYAML. Load params via `utils.load_params()`.
2. For each book across all genres:
   - Validate `gutenberg_id` is a positive integer (path traversal prevention).
   - Check if `data/raw/{gutenberg_id}.txt` already exists and is non-empty. If so, skip with log: `"Skipping {title}: already downloaded"`.
   - Download using gutenbergpy: `raw = textget.get_text_by_id(gutenberg_id)` then `clean = textget.strip_headers(raw)`.
   - Decode: `text = clean.decode('utf-8', errors='replace')`.
   - Validate: text length > 1000 characters. If not, log warning per D-12 and continue.
   - Save to `data/raw/{gutenberg_id}.txt`.
   - Print verbose timing per D-06: `"[1/15] Dracula (345): downloaded... done (2.3s)"`.
   - Sleep `params['corpus']['download_sleep']` seconds between downloads (default 2s).
3. At the end, print summary: `"Downloaded: 15/15 books (0 failures)"` or with failure count.
4. If any download fails, log which books failed but do NOT abort (per D-12).

**CLI interface:** `argparse` with optional `--download-sleep N` override.

**Import pattern:**
```python
from gutenbergpy.textget import get_text_by_id, strip_headers
```

</implementation>

### Task 01-01-02: Preprocess text -- tokenize, normalize, filter stopwords

**Goal:** Implement `scripts/02_preprocess.py` that reads raw text from `data/raw/`, tokenizes, lowercases, removes punctuation, filters NLTK English stopwords, enforces the 10,000 unique word minimum (D-13), and saves processed books as JSON word lists to `data/processed/{gutenberg_id}.json`.
**Requires:** Task 01-01-01 (downloaded raw text in `data/raw/`)

<implementation>

**Create `scripts/02_preprocess.py`:**

1. Load params via `utils.load_params()`. Load `corpus/books.yaml` for book metadata.
2. Download NLTK stopwords if not present: `nltk.download('stopwords', quiet=True)`.
3. For each book (by iterating `data/raw/*.txt`):
   - Read text from `data/raw/{gutenberg_id}.txt`.
   - Tokenize: use `re.findall(r"[a-z]+", text.lower())` -- this lowercases, removes punctuation, and splits in one step. Do NOT use NLTK tokenizer (overkill for this use case).
   - Remove stopwords: filter against `nltk.corpus.stopwords.words('english')` set.
   - Compute unique word count: `len(set(tokens))`.
   - If unique words < `params['corpus']['min_unique_words']` (default 10,000): log warning per D-13 `"Skipping {title}: only {N} unique words after filtering (minimum 10,000)"` and skip.
   - Save to `data/processed/{gutenberg_id}.json` as: `{"gutenberg_id": id, "title": "...", "author": "...", "genre": "...", "tokens": [...], "unique_word_count": N, "total_token_count": M}`.
   - Print verbose timing per D-06: `"[1/15] Dracula (345): {N} unique words, {M} total tokens... done (0.8s)"`.
4. Print summary: number of books processed, number skipped, per-genre counts.
5. Per D-12: if a genre has fewer than 5 books after filtering, print warning: `"Genre {genre}: only {N} books available (expected 5) -- results may be less reliable"`.

**CLI interface:** `argparse` with optional `--min-unique-words N` override.

</implementation>
