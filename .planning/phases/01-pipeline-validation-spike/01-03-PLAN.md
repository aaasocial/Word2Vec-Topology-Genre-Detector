# Plan 01-03: Persistent Homology, Classification, and Validation

**Phase:** 01 -- Pipeline Validation Spike
**Goal:** Compute weighted Vietoris-Rips persistent homology per book, convert diagrams to persistence images, build feature vectors (persistence images + cluster distribution), train SVM with LOOCV, run permutation test, produce GO/NO-GO verdict and validation report. Also implement the benchmark script for timing Vietoris-Rips at various word counts.
**Requirements:** HOM-01, HOM-02, HOM-03, HOM-04, HOM-05, HOM-06, HOM-07, HOM-08, VALID-01, VALID-02, VALID-03
**Status:** complete

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Computational explosion in Vietoris-Rips (O(n^d) simplices) | Medium | max_words cap (default 500); auto-timeout-retry reducing by 100 per D-11; hard floor at 100 words |
| Data leakage: PCA/scaler fit on test fold | Medium | Wrap PCA + StandardScaler + SVM in sklearn Pipeline; LOOCV via permutation_test_score handles fold splitting |
| Silent math errors from non-metric weighted distance matrix | Low | Documented: d/(w_i+w_j) violates triangle inequality but giotto-tda Ripser accepts precomputed matrices; validation spike confirms empirical behavior |
| Overfitting: 450D features / 15 samples | High | PCA to 20 components (per params.yaml); permutation test detects chance-level performance; GO/NO-GO gate rejects spurious results |
| Degenerate persistence images from empty diagrams | Low | Handle empty diagrams by passing `np.zeros((0, 2))` to persim; result is zero-valued image (no topology detected) |
</threat_model>

### Task 01-03-01: Persistent homology computation

**Goal:** Implement `scripts/04_compute_homology.py` that loads per-book vectors and TF-IDF weights, constructs the weighted distance matrix `d(i,j)/(w_i+w_j)`, computes Vietoris-Rips persistent homology using giotto-tda with `metric='precomputed'`, implements timeout-retry logic (D-11), and saves persistence diagrams.
**Requires:** Task 01-02-01 (vectors and TF-IDF weights in `data/features/`), Task 01-03-00 (test stubs)

<implementation>

**Create `scripts/04_compute_homology.py`:**

**Step 1: Load per-book data**
- For each book, load `data/features/vectors_{id}.npy` and `data/features/tfidf_{id}.npy`.
- Load params for `max_words`, `timeout`, `retry_step`, `min_words`, `homology_dimensions`, `epsilon_max`.

**Step 2: Weighted distance matrix (per HOM-01, VALID-02)**
Implement `compute_weighted_distance_matrix(vectors, tfidf_weights)` exactly as in RESEARCH.md Q1:
```python
from scipy.spatial.distance import pdist, squareform

raw_dist = squareform(pdist(vectors, metric='euclidean'))
weight_sums = tfidf_weights[:, None] + tfidf_weights[None, :]
weight_sums = np.maximum(weight_sums, 1e-10)
weighted_dist = raw_dist / weight_sums
np.fill_diagonal(weighted_dist, 0.0)
```

**Step 3: Vietoris-Rips with timeout-retry (per D-11)**
- Use `VietorisRipsPersistence(metric='precomputed', homology_dimensions=(0, 1), max_edge_length=epsilon_max, n_jobs=1)`.
- Input shape: `weighted_dist[np.newaxis, :, :]` -- shape `(1, n_words, n_words)`.
- **Timeout implementation on Windows:** Use `multiprocessing.Process` with a timeout (cannot use `signal.alarm` on Windows). Wrap `VR.fit_transform()` in a subprocess. If it exceeds `timeout` seconds:
  - Log per D-11: `"Book {title} timed out at {max_words} words -- retrying at {max_words - retry_step} words"`.
  - Reduce max_words by `retry_step` (default 100), re-select top words, rebuild distance matrix, retry.
  - Continue until success or max_words < min_words (default 100), then skip with warning.
- On success, save diagram to `data/features/diagrams_{gutenberg_id}.npy`.

**Step 4: Verbose output per D-06**
- Print: `"[1/15] Dracula (345): {N} words, computing homology... done ({t}s)"`.
- If retried: `"[1/15] Dracula (345): timed out at 500 words, retried at 400... done ({t}s)"`.
- At end: summary of all books, any skipped.

**CLI interface:** `argparse` with `--max-words`, `--timeout`, `--epsilon-max` overrides.

### Task 01-03-02: Feature engineering -- persistence images, clustering, and concatenation

**Goal:** Implement `scripts/05_build_features.py` that converts persistence diagrams to persistence images via persim, runs k-means clustering on all word vectors, computes per-book cluster distribution weighted by TF-IDF, concatenates features with alpha weighting, applies normalization, and saves final feature vectors.
**Requires:** Task 01-03-01 (persistence diagrams in `data/features/`), Task 01-02-01 (Word2Vec model, vectors, TF-IDF weights)

<implementation>

**Create `scripts/05_build_features.py`:**

**Step 1: Convert giotto-tda diagrams to persim format (per HOM-03)**
Implement `giotto_to_persim(diagrams, dim)` exactly as in RESEARCH.md Q5:
```python
def giotto_to_persim(diagrams, dim):
    diag = diagrams[0]
    mask = diag[:, 2] == dim
    bd_pairs = diag[mask, :2]
    valid = bd_pairs[:, 0] < bd_pairs[:, 1]
    bd_pairs = bd_pairs[valid]
    finite = np.isfinite(bd_pairs[:, 1])
    bd_pairs = bd_pairs[finite]
    if len(bd_pairs) == 0:
        return np.zeros((0, 2))
    return bd_pairs
```

**Step 2: Compute persistence images (per HOM-03)**
- Collect all H0 and H1 diagrams across all books (in persim format).
- Create PersistenceImager:
```python
pimgr = PersistenceImager(
    pixel_size=0.05,  # will be adjusted after fit
    weight='persistence',
    kernel='gaussian',
    kernel_params={'sigma': params['features']['sigma']},
)
```
- Fit on all diagrams together (critical: consistent coordinate ranges across books).
- After fitting, adjust `pixel_size` to achieve the target `grid_resolution` (default 20):
```python
birth_span = pimgr.birth_range[1] - pimgr.birth_range[0]
pers_span = pimgr.pers_range[1] - pimgr.pers_range[0]
pimgr.pixel_size = max(birth_span, pers_span) / params['features']['grid_resolution']
```
- Transform each book's diagram. Pass `skew=True` to convert (birth, death) to (birth, persistence).
- Flatten each image to a 1D vector.
- Handle empty diagrams: result is a zero vector of the correct length.
- Print per D-06: `"Persistence images: H0 ({M}x{M}={F} features), H1 ({M}x{M}={F} features)... done ({t}s)"`.

**Step 3: K-means clustering (per HOM-04)**
- Load the Word2Vec model. Get all word vectors (L2-normalized).
- Run k-means: `KMeans(n_clusters=params['features']['k_clusters'], random_state=42, n_init=10).fit(all_vectors)`.
- Save cluster assignments.
- Print: `"K-means clustering: {K} clusters on {V} words... done ({t}s)"`.

**Step 4: Per-book cluster distribution (per HOM-05)**
For each book:
- Load the book's word list and TF-IDF weights.
- For each word, find its cluster assignment.
- Compute the K-dimensional distribution: sum TF-IDF weights per cluster.
- L2-normalize the distribution vector.

**Step 5: Concatenate features (per HOM-06)**
For each book:
- L2-normalize the H0 persistence image vector.
- L2-normalize the H1 persistence image vector.
- L2-normalize the cluster distribution vector.
- Concatenate with alpha weighting: `alpha * np.concatenate([h0_img, h1_img]) + (1 - alpha) * cluster_dist` is NOT correct because dimensions differ. Instead:
  - `structure_vec = np.concatenate([h0_norm, h1_norm])` -- topology part
  - `location_vec = cluster_dist_norm` -- cluster part
  - Pad to concatenate: `feature = np.concatenate([alpha * structure_vec, (1 - alpha) * location_vec])`
- Save to `data/features/features_{gutenberg_id}.npy`.
- Print per D-06: `"[1/15] Dracula (345): feature vector ({D}D)... done ({t}s)"`.

**Step 6: Save combined feature matrix**
- Stack all feature vectors into `data/features/feature_matrix.npy` -- shape `(15, D)`.
- Save labels to `data/features/labels.npy` -- shape `(15,)`.
- Save book order to `data/features/book_order.json`.

**CLI interface:** `argparse` with `--grid-resolution`, `--sigma`, `--k-clusters`, `--alpha` overrides.

</implementation>
