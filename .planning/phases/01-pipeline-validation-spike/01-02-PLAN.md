# Plan 01-02: Word2Vec Training and TF-IDF Computation

**Phase:** 01 -- Pipeline Validation Spike
**Goal:** Train a single shared skip-gram Word2Vec model on the entire preprocessed corpus, compute per-book TF-IDF weights using corpus-level IDF (no genre labels in IDF computation), and construct per-book weighted point clouds (word vectors + TF-IDF weights) saved as numpy arrays.
**Requirements:** PIPE-03, PIPE-04, PIPE-05

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Non-deterministic Word2Vec training producing irreproducible results | Low | `workers=1` and `seed=42` per params.yaml; deterministic training |
| Genre labels leaking into IDF computation | Medium | TfidfVectorizer fits on ALL books at once with no genre grouping; no genre labels are passed to sklearn |
| OOV words between Word2Vec and TF-IDF vocabularies | Low | Restrict TfidfVectorizer vocabulary to Word2Vec vocabulary via `vocabulary=` parameter |
| Resource exhaustion during training on large corpus | Low | Corpus is only 15 books (~750k tokens); Word2Vec with 100D completes in seconds |
</threat_model>

### Task 01-02-01: Train Word2Vec and compute TF-IDF

**Goal:** Implement `scripts/03_train_embeddings.py` that trains skip-gram Word2Vec on the full preprocessed corpus, computes per-book TF-IDF weights using scikit-learn TfidfVectorizer fitted on all books (no genre labels), selects top-K words per book by TF-IDF, constructs weighted point clouds, and saves all artifacts.
**Requires:** Task 01-01-02 (preprocessed JSON in `data/processed/`), Task 01-02-00 (test stubs)

<implementation>

**Create `scripts/03_train_embeddings.py`:**

**Step 1: Load preprocessed corpus**
- Read all `data/processed/{gutenberg_id}.json` files.
- Build `all_books_tokens`: list of token lists (one per book).
- Build `book_metadata`: list of dicts with `gutenberg_id`, `title`, `genre`.
- Print: `"Loaded {N} books, {M} total tokens"`.

**Step 2: Train Word2Vec (per D-05, PIPE-03)**
- Split each book's tokens into chunks of 1000 words to create "sentences" for Word2Vec context windows.
- Train with gensim:
```python
model = Word2Vec(
    sentences=corpus_sentences,
    vector_size=params['word2vec']['vector_size'],  # default 100
    window=params['word2vec']['window'],              # default 5
    min_count=params['word2vec']['min_count'],        # default 5
    sg=params['word2vec']['sg'],                      # 1 = skip-gram
    epochs=params['word2vec']['epochs'],              # default 10
    workers=params['word2vec']['workers'],            # 1 = deterministic
    seed=params['word2vec']['seed'],                  # 42
    negative=5,
)
```
- Save model: `model.save('data/models/word2vec.model')`.
- Print verbose timing per D-06: `"Training Word2Vec ({vocab_size} words, {vector_size}D)... done ({t}s)"`.

**Step 3: Compute TF-IDF (per PIPE-04)**
- Rejoin each book's tokens into a single string.
- Fit TfidfVectorizer on ALL books at once (critical: no genre labels):
```python
vectorizer = TfidfVectorizer(
    sublinear_tf=True,
    smooth_idf=True,
    norm=None,
    use_idf=True,
    lowercase=False,
    token_pattern=r'(?u)\b\w+\b',
    vocabulary=list(model.wv.key_to_index.keys()),
)
tfidf_matrix = vectorizer.fit_transform(book_texts)
feature_names = vectorizer.get_feature_names_out()
```
- Print: `"TF-IDF computed: {vocab_size} features across {n_books} books... done ({t}s)"`.
- Log OOV count: words in preprocessed vocab but filtered by Word2Vec `min_count`.

**Step 4: Construct per-book weighted point clouds (per PIPE-05)**
For each book:
- Extract TF-IDF row for this book.
- Select top `max_words` words by TF-IDF weight (default 500 from params).
- For each selected word, get its L2-normalized vector: `model.wv.get_vector(word, norm=True)`.
- Save vectors as `data/features/vectors_{gutenberg_id}.npy` -- shape `(n_words, embedding_dim)`.
- Save TF-IDF weights as `data/features/tfidf_{gutenberg_id}.npy` -- shape `(n_words,)`.
- Save word list as `data/features/words_{gutenberg_id}.json` -- for debugging/traceability.
- Print per D-06: `"[1/15] Dracula (345): top {N} words selected, vectors saved... done ({t}s)"`.

**Step 5: Save TF-IDF vectorizer**
- Save fitted vectorizer via joblib to `data/models/tfidf_vectorizer.joblib` for potential reuse.

**CLI interface:** `argparse` with overrides for `--vector-size`, `--window`, `--min-count`, `--max-words`.

</implementation>
