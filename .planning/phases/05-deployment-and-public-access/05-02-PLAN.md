  <action>
**Create a tarball of runtime model files + pre-built cache for download during Docker build.**

The tarball must contain exactly:
```
data/models/word2vec_w15.model
data/models/tfidf_vectorizer_w15.joblib
data/models/kmeans_w15_k200.pkl
data/models/svm_pipeline.joblib
data/models/persistence_imager.joblib
data/models/genre_names.json
data/cache/   (entire directory -- all .json and .npy files)
```

Do NOT include other window variants (w5, w10, w20) or other k-cluster variants -- only the production defaults (window=15, k=200).

Step 1: Create the tarball locally:
```bash
cd "C:/Users/Eason/Desktop/CC/Word2Vec Genre Analyser"
tar czf genre-topology-data.tar.gz \
  data/models/word2vec_w15.model \
  data/models/tfidf_vectorizer_w15.joblib \
  data/models/kmeans_w15_k200.pkl \
  data/models/svm_pipeline.joblib \
  data/models/persistence_imager.joblib \
  data/models/genre_names.json \
  data/cache/
```

Verify the tarball:
```bash
tar tzf genre-topology-data.tar.gz | head -20
ls -lh genre-topology-data.tar.gz  # Should be ~200-400MB
```

Step 2: Create GitHub Release using gh CLI:
```bash
gh release create v1.0-data \
  --title "v1.0 Runtime Data" \
  --notes "Pre-built model files and cache for Docker deployment. Download URL for RELEASE_URL build arg." \
  genre-topology-data.tar.gz
```

If `gh` is not authenticated, the executor should print the instructions for manual release creation and pause. The user can create the release manually via the GitHub web UI (Releases -> Create new release -> attach genre-topology-data.tar.gz).

Step 3: Get the download URL for the tarball asset:
```bash
gh release view v1.0-data --json assets -q '.assets[0].url'
```

The URL format will be something like:
`https://github.com/{owner}/{repo}/releases/download/v1.0-data/genre-topology-data.tar.gz`

Record this URL -- it will be needed as RELEASE_URL for Railway deployment.

Step 4: Clean up local tarball:
```bash
rm genre-topology-data.tar.gz
```

The user must perform the following steps in the Railway dashboard (Claude cannot access Railway):

**Step 1: Create Railway project**
1. Go to https://railway.app/dashboard
2. Click "New Project" -> "Deploy from GitHub Repo"
3. Select the Word2Vec Genre Analyser repository
4. Railway will detect the Dockerfile and begin building

**Step 2: Add Redis addon**
1. In the Railway project, click "New" -> "Database" -> "Add Redis"
2. Railway automatically injects REDIS_URL into the service environment

**Step 3: Set RELEASE_URL build variable**
1. Click on the app service -> "Variables"
2. Add a new variable: `RELEASE_URL` = the GitHub Release download URL from Task 1
   (Format: `https://github.com/{owner}/{repo}/releases/download/v1.0-data/genre-topology-data.tar.gz`)
3. This triggers a new build that downloads the model data during Docker build

**Step 4: Verify deployment**
1. Wait for the build to complete (may take 5-10 minutes)
2. Railway assigns a public URL (visible in the service settings under "Networking" -> "Public Networking")
3. Generate a domain if none exists: Settings -> Networking -> Generate Domain
4. Visit the public URL -- you should see the application

**Step 5: Report back**
Tell Claude:

- Whether the build succeeded
- Whether the app loads in the browser
