1. **Create `backend/pipeline/precompute_vr.py`**:
   - Follow the existing `precompute_viz.py` pattern (imports, logging, cache usage).
   - Import `build_weighted_distance_matrix` from `backend.pipeline.homology`.
   - Main function `precompute_vr_edges(genre, words, vectors, tfidf_weights, epsilon_max, projection_coords)`:
     - Build weighted distance matrix using existing `build_weighted_distance_matrix`.
     - Extract upper triangle: for each pair (i, j) where i < j, if `dist_matrix[i,j] <= epsilon_max`, emit edge `[i, j, round(dist_matrix[i,j], 5)]`.
     - Sort edges by eps_birth ascending (critical for drawRange optimization in browser).
     - Return `{words: list[str], edges: list[list], epsilon_max: float, positions: list[list]}`.
   - Limit to top 500 words by TF-IDF weight per genre (per CORPUS-03 max_words cap) to keep payload tractable.
   - Entry point `precompute_all_vr(window, projection, force)`:
     - Load Word2Vec model, books.yaml, TF-IDF data (same pattern as precompute_viz.py).
     - For each genre: select top-500 TF-IDF words, get their vectors and projection coords.
     - Call `precompute_vr_edges`, cache result as `vr_edges:{genre}:{projection}:{window}`.
     - Also support per-book: cache as `vr_edges_book:{gutenberg_id}:{projection}:{window}`.
   - Add `get_cached_vr_edges(genre, projection, window)` helper.
   - Call from `precompute_viz.py` main flow or as standalone: `python -m backend.pipeline.precompute_vr`.

2. **Create `backend/pipeline/tests/test_precompute_vr.py`**:

   - Assert epsilon_max=0 returns empty edges.

1. **Create `vrFiltering.ts`**:
   - Export `filterEdgesByEpsilon(edges, epsilon, positions, birthWindow = 0.005)`:
     - Edges are `[word_a_idx, word_b_idx, eps_birth][]`, PRE-SORTED by eps_birth.
     - Since edges are sorted, use binary search to find the cutoff index where `eps_birth > epsilon`. Only iterate up to that index (early exit optimization).
     - Pre-allocate `Float32Array(maxEdges * 6)` for positions and colors.
     - For each visible edge: copy start/end vertex positions from the positions array.
     - For edge coloring: if `|eps_birth - epsilon| < birthWindow`, use birth highlight color (#FACC15 -> r=0.98, g=0.80, b=0.08). Otherwise use default edge color (#4A4A5A -> r=0.29, g=0.29, b=0.35).
     - Return `{linePositions: Float32Array, lineColors: Float32Array, count: number}`.
   - Export `getVisibleEdgeCount(edges, epsilon)`: binary search for count (used by UI to show edge count label).
   - CRITICAL: No server calls. This runs in the render loop at 60fps.

2. **Create `vrFiltering.test.ts`**: Unit tests for filterEdgesByEpsilon and getVisibleEdgeCount.
   - Test with 4 edges at eps_birth [0.2, 0.5, 0.8, 1.5], epsilon=1.0 -> 3 visible.
   - Test birth highlight: edge at eps_birth=1.0, epsilon=1.0 -> highlighted color.
   - Test epsilon=0 -> 0 visible.
   - Test all edges visible when epsilon >= max eps_birth.

3. **Create `useVRData.ts`**: React Query hook.
   ```typescript
   export function useVRData(genre: string | null, projection: ProjectionKey)
   ```
   - Query key: `['vr-edges', genre, projection]`.
   - Fetches from `/viz/vr/${genre}?projection=${projection}`.
   - `staleTime: Infinity` (precomputed).
   - `enabled: !!genre`.
   - Returns typed `VRPayload` data.

4. **Create `VREdges.tsx`**: R3F component using `THREE.LineSegments`.
   - Props: `edges`, `positions` (from VR payload), `epsilon` (from store).
   - Creates `THREE.BufferGeometry` with pre-allocated position and color attributes sized for max edge count.
   - In `useFrame`: read `vrEpsilon` from `useVisualizationStore.getState()` (hot-path, no subscription).
   - Call `filterEdgesByEpsilon` to get visible line positions and colors.
   - Update geometry attributes: set position array, color array, and `geometry.setDrawRange(0, count * 2)` (2 vertices per line segment).
   - Mark `.needsUpdate = true` on both attributes.
   - Material: `THREE.LineBasicMaterial({ vertexColors: true, transparent: true, opacity: 0.7 })`.
   - Birth/death fade: track `highlightTimestamps` map. When an edge transitions from not-visible to visible (birth), record timestamp. In useFrame, edges within 500ms of birth get highlight color; after 500ms, fade to default over 300ms. Use lerp on the color buffer.

5. **Create `VRViewer.tsx`**: R3F Canvas wrapper for VR panel.
   - Separate `<Canvas>` instance from scatter (per CONTEXT.md).
   - Props: none (reads from store).
   - Reads `selectedGenre`, `projection` from store.
   - Uses `useVRData(selectedGenre, projection)`.
   - Renders word points using simplified point rendering (all points at 40% opacity, selected genre at 80%). Use `THREE.Points` with basic ShaderMaterial (simpler than scatter PointCloud — no TF-IDF brightness needed here).
   - Renders `<VREdges>` with loaded edge data.
   - Own `<OrbitControls>` (independent camera from scatter).
   - Canvas props: `gl={{ antialias: true, alpha: false, preserveDrawingBuffer: true }}` (preserveDrawingBuffer for PNG export in Plan 03).
   - Camera: PerspectiveCamera fov=60.
   - Background color: `#0A0A0F`.
   - Empty state: show points with no edges + text overlay "Drag the epsilon slider to explore filtration" when epsilon=0.

6. **Create `EpsilonSlider.tsx`** (per TOPO-04):
   - shadcn Slider or custom range input.
   - Label: "Filtration radius (epsilon)" + current value in mono font.
   - Range: 0 to `epsilon_max` from VR data. Step: `epsilon_max / 200`.
   - Track: `#2A2A3A` bg, `#FACC15` fill.
   - Thumb: 16px circle, `#F5F5FF` fill, `#FACC15` border.
   - On change: `setVrEpsilon(value)` in store. NO server calls. NO debounce (render loop reads directly).
   - Show edge count: "{N} edges visible" below slider in muted text.
   - `aria-label="Filtration radius epsilon"`.

7. **Wire into `TopologyPanel.tsx`**: Replace right panel VR placeholder with `<VRViewer />` + `<EpsilonSlider />` below it.
