**Step A — split finite from infinity AT THE TOP of the draw block, before any axis math:**

```ts
const FINITE = (v: number) => Number.isFinite(v)
const rawPts = data?.points ?? []
const finitePts = rawPts.filter(([b, d]) => FINITE(b) && FINITE(d) && d > b)
const infinityPts = rawPts.filter(([b, d]) => FINITE(b) && !FINITE(d))
```

**Step B — compute axis bounds from finite only:**

```ts
const finiteVals = finitePts.flatMap(([b, d]) => [b, d])
const dataMax = finiteVals.length > 0 ? Math.max(...finiteVals) : data.epsilon_max
const axisMax = Math.min(dataMax * 1.1, data.epsilon_max)
```

The infinity points must NOT enter `Math.max` — that's the root cause of the auto-rescale bug.

**Step C — sqrt dot scaling for finite dots (use these EXACT identifiers and bounds):**

```ts
const BASE_RADIUS = 1.5        // minimum visible size, even at near-zero persistence
const RADIUS_SCALE = 5.0       // multiplier for the sqrt curve
const MAX_RADIUS = 6.5         // cap so a single outlier doesn't dominate
const finitePersistences = finitePts.map(([b, d]) => d - b)
const maxFinitePersistence = finitePersistences.length > 0
  ? Math.max(...finitePersistences)
  : 1.0  // avoid divide-by-zero on empty / all-infinity case

for (const [birth, death] of finitePts) {
  const persistence = death - birth
  const normalized = persistence / maxFinitePersistence   // in [0, 1]
  const radius = Math.min(
    MAX_RADIUS,
    BASE_RADIUS + RADIUS_SCALE * Math.sqrt(normalized)
  )
  const cx = tx(birth)
  const cy = ty(death)
  ctx.beginPath()
  ctx.arc(cx, cy, radius, 0, Math.PI * 2)
  ctx.fill()
}
```

Use the existing `ctx.fillStyle = '#FACC15'` and the existing `globalAlpha` heuristic if you keep that.
