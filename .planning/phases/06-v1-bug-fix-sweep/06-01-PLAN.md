**File 1: `.githooks/pre-commit` (POSIX `#!/bin/sh`)**

Exact behavior:

1. **0-byte rejection:** For each staged file matching `.planning/**/*.md`, if its size on disk is 0 bytes, print a clear error and `exit 1`. Use `git diff --cached --name-only --diff-filter=AM` to list staged adds/modifies, then `wc -c` (or `stat`) per file.
2. **Snapshot:** After the 0-byte check passes, copy `.planning/ROADMAP.md`, `.planning/STATE.md`, and `.planning/PROJECT.md` (each only if it exists) to `.planning/.snapshots/$(date -u +%Y-%m-%dT%H%M%SZ)/`. Create the timestamped subdir with `mkdir -p`. Use UTC timestamps to avoid cross-machine drift.
3. **Exit 0** on success. Snapshot failure is non-fatal (`|| true`) — we never want to block a commit because snapshotting hiccupped, but the 0-byte check IS fatal.

Use literal content:

```sh
#!/bin/sh
# Pre-commit hook installed via scripts/install-hooks.sh.
# Purpose:
#   1. Reject any 0-byte commit to .planning/**/*.md (BUG-04, PITFALLS.md §15).
#   2. Snapshot ROADMAP.md / STATE.md / PROJECT.md to .planning/.snapshots/{UTC-timestamp}/
#      so a future wipe is always one cp away from recovery.
set -eu

REPO_ROOT=$(git rev-parse --show-toplevel)
cd "$REPO_ROOT"

# --- 1. Reject 0-byte planning markdown ---
FAILED=0
STAGED=$(git diff --cached --name-only --diff-filter=AM | grep -E '^\.planning/.*\.md$' || true)
for f in $STAGED; do
  if [ -e "$f" ]; then
    SIZE=$(wc -c < "$f" | tr -d ' ')
    if [ "$SIZE" = "0" ]; then
      echo "ERROR: Refusing to commit 0-byte planning file: $f"
      echo "       (PITFALLS.md §15 — restore from .planning/.snapshots/ or revert the change)"
      FAILED=1
    fi
  fi
done
if [ "$FAILED" -ne 0 ]; then
  exit 1
fi

# --- 2. Snapshot the load-bearing planning docs ---
TS=$(date -u +%Y-%m-%dT%H%M%SZ)
SNAP_DIR=".planning/.snapshots/$TS"
mkdir -p "$SNAP_DIR" 2>/dev/null || true
for f in .planning/ROADMAP.md .planning/STATE.md .planning/PROJECT.md; do
  if [ -f "$f" ]; then
    cp "$f" "$SNAP_DIR/$(basename "$f")" 2>/dev/null || true
  fi
done

exit 0
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local dev → repo history | Pre-commit hook + installer run in the user's local environment; outputs land in `.git/config` (local-only) and the working tree (`.githooks/`, `.gitattributes`, `scripts/install-hooks.sh`) |
| CI runner → repo | GitHub Actions runner reads `.planning/**/*.md` via `actions/checkout@v4` and runs `find`/`wc` against them; no untrusted input enters the check |
| Foreign clone → hook script | Malicious actor with commit access could modify `.githooks/pre-commit` to leak or skip checks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Tampering | `.githooks/pre-commit` | mitigate | Hook is versioned in-repo; tampering visible in `git diff`. CI backstop (`planning-files-check.yml`) enforces same 0-byte check independent of the local hook — bypassing the local hook does NOT bypass CI. |
| T-06-01-02 | Denial of Service | snapshot directory `.planning/.snapshots/` | accept | Snapshot grows by 3 files (~30KB total) per commit. Worst case after 1000 commits: ~30MB local. Acceptable; directory is gitignored so no remote impact. User can `rm -rf` at will. |
| T-06-01-03 | Repudiation | snapshot timestamps | accept | UTC timestamps from `date -u`. No cryptographic signing — snapshots are recovery artifacts, not audit trail. If audit trail is required, `git reflog` already covers it. |
| T-06-01-04 | Information Disclosure | snapshot files contain entire ROADMAP/STATE/PROJECT | N/A | Files are already public-readable in the repo working tree; snapshot is a sibling copy, no new exposure. |
| T-06-01-05 | Elevation of Privilege | `scripts/install-hooks.sh` runs `git config core.hooksPath .githooks` | mitigate | Config is repo-local (no `--global`). Cannot affect other repos. Script does not run with sudo / does not need elevated permissions. |
| T-06-01-06 | Tampering | `.gitattributes` LFS-exclusion bypass | mitigate | Even if a contributor removes the `-lfs` line, the pre-commit hook still rejects 0-byte commits — defense in depth. CI backstop independent of `.gitattributes`. |
